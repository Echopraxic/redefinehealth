## вңЁ What is Healthspan OS?

> **Healthspan OS runs as a background service on macOS.** It reads your iMessage inbox in real time, routes messages from registered users through a command system, and responds with personalized health guidance generated by Claude (Anthropic). 

**No app to download. No account to create. It works through iMessage.**

рҹ”№ *Also proactively messages users вҖ” daily supplement reminders, peptide injection prompts, protocol check-ins вҖ” all timed to each user's individual schedule.*

## рҹҺҜ Features

### рҹ§¬ Protocol Engine

| Protocol | Capabilities |
|----------|--------------|
| **Supplement** | Daily reminders at wake/sleep times, dose detection, streak tracking |
| **Peptide** | Full injection cycle management вҖ” pre-reminder, injection prompt, post-check-in, cycle-end вҖ” with BAC water reconstitution guides |
| **Skin** | Morning stack, midday SPF, evening routine, weekly photo check-in (Sundays), biweekly collagen review |
| **Sleep** | Morning light anchor, 2pm caffeine cutoff, pre-sleep stack (1h before), wind-down protocol (30min before), sleep quality check-ins |
| **Longevity** | NAD+ stack timing, CoQ10 with meals, Berberine before carbs, fasting tracking, monthly bloodwork (1st), Rapamycin weekly (Saturdays), quarterly senolytics |

рҹ”№ *Protocols auto-activate from each user's `goals` array.*

<br/>

### рҹӨ– AI Integration (Claude + Prompt Caching)

**Two-layer prompt caching architecture:**

1. **Layer 1** вҖ” Base system prompt (`cache_control: ephemeral`)
   - Health assistant persona, safety rules, response style

2. **Layer 2** вҖ” Domain-specific templates per command
   - Progress analysis, side effects, reconstitution, skin coaching, sleep coaching, onboarding

рҹ”№ *Only user message + context are uncached per request вҖ” minimal latency & API costs*

**AI-powered flows:**
- General health Q&A
- Progress report analysis (30-day compliance + protocol adherence)
- Side effect correlation (severity-gated вҖ” emergencies bypass AI)
- Peptide reconstitution Q&A
- Skin protocol coaching (weekly ratings, goal-specific tips)
- Sleep coaching (HRV interpretation, circadian science)
- Personalized welcome messages on onboarding

<br/>

### рҹ“Ҡ Compliance Tracking

| Feature | Implementation |
|---------|---------------|
| **Dose logging** | Every taken/skipped dose вҶ’ `compliance_log` |
| **Injection tracking** | Every peptide injection вҶ’ `peptide_injections` |
| **Streak calculation** | `calculateStreak()` вҖ” current/best per supplement |
| **Report generation** | `generateReport()` вҖ” 30-day trends + trajectory analysis |
| **User queries** | `status` (full report) or `streak` (supplement breakdown) |

<br/>

### рҹӣЎпёҸ Safety Layer

**Interaction checker** вҖ” Runs on every `registerUser()` call
- Checks user's full stack (supplements + peptides) against **18 known interactions**
- Known interactions include: Magnesium вҶ” Zinc competition, Fish Oil + high-dose Vitamin E bleeding risk, Berberine + Metformin hypoglycemia potentiation, NAD+ precursors + SIRT1 activators dose timing, and 14 others
- вҡ пёҸ Warnings sent before first reminder

**Side effect handler** вҖ” Severity classification (regex-based, pre-LLM)
- рҹ”ҙ `emergency` (chest pain, difficulty breathing, anaphylaxis) вҶ’ immediate 911, no AI delay
- рҹҹ  `severe` (severe allergic reaction, extreme dizziness) вҶ’ urgent escalation
- рҹҹЎ `moderate` вҶ’ AI analysis with dose correlation
- рҹҹў `mild` вҶ’ standard AI analysis

<br/>

### рҹ‘Ҙ Multi-User Platform

Every feature is scoped per user:
- Each user вҶ’ their own row in `users` table
- Reminders scheduled per user with timezone + wake/sleep times
- Protocol activation per-user based on goals
- Compliance logs keyed by `userId`
- Conversation flow state keyed by phone number

<br/>

### рҹ”Ң Multi-Step Onboarding

Unknown senders who message `start`, `hi`, `hello`, or `join` are offered onboarding. 

**4-step conversation flow:**
1. Name
2. Goals (longevity / sleep / skin / energy / cognition / body-composition)
3. Wake time
4. Sleep time

рҹ”№ *State held in `ConversationFlowManager` (in-memory, 30-minute TTL)*

<br/>

| `safetyGuardPlugin` | Blocks sends containing dangerous content patterns |
