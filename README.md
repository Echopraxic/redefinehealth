# Healthspan OS

> iMessage-native precision wellness assistant — multi-user, AI-powered, protocol-driven

This monorepo contains two packages:

| Package | Description |
|---------|-------------|
| [`@photon-ai/imessage-kit`](./src/) | Type-safe macOS iMessage SDK (TypeScript) |
| [`healthspan-os`](./app/) | Multi-user wellness assistant built on the SDK |

---

## What is Healthspan OS?

Healthspan OS runs as a background service on macOS. It reads your iMessage inbox in real time, routes messages from registered users through a command system, and responds with personalized health guidance generated by Claude (Anthropic). It also proactively messages users — daily supplement reminders, peptide injection prompts, protocol check-ins — all timed to each user's individual schedule.

No app to download. No account to create. It works through iMessage.

## Features

### Protocol Engine

| Protocol | What it does |
|----------|-------------|
| **Supplement** | Daily reminders at each user's wake/sleep time, dose detection, streak tracking |
| **Peptide** | Full injection cycle management — pre-reminder, injection prompt, post-check-in, cycle-end — with BAC water reconstitution guides |
| **Skin** | Morning stack reminder, midday SPF nudge, evening routine, weekly photo check-in (Sundays), biweekly collagen review |
| **Sleep** | Morning light anchor, 2pm caffeine cutoff, pre-sleep stack (1h before bed), wind-down protocol (30min before bed), sleep quality check-ins |
| **Longevity** | NAD+ stack timing, CoQ10 with meals, Berberine before carb meals, fasting window tracking, monthly bloodwork reminder (1st of month), Rapamycin weekly tracking (Saturdays), quarterly senolytic reminder |

Protocols are **auto-activated** from each user's `goals` array. A user with `['sleep', 'skin']` automatically gets SleepProtocol and SkinProtocol scheduled on startup.

### AI Integration (Claude + Prompt Caching)

All AI responses use two-layer prompt caching:
1. **Layer 1** — base system prompt cached with `cache_control: ephemeral` (health assistant persona, safety rules, response style)
2. **Layer 2** — domain-specific template cached per command type (progress analysis, side effects, reconstitution, skin coaching, sleep coaching, onboarding)

This means only the user's message + context are uncached per request — keeping latency low and API costs minimal.

Supported AI-powered flows:
- General health Q&A
- Progress report analysis (30-day compliance trends + protocol adherence)
- Side effect correlation (maps symptoms to recent doses; severity-gated — emergencies bypass AI for immediate escalation)
- Peptide reconstitution Q&A
- Skin protocol coaching (weekly ratings, goal-specific tips)
- Sleep coaching (HRV interpretation, circadian science)
- Personalized welcome messages on onboarding

### Compliance Tracking

- Every dose taken or skipped is written to SQLite (`compliance_log` table)
- Every peptide injection is logged separately (`peptide_injections` table)
- `calculateStreak()` computes per-supplement streaks and current/best run lengths
- `generateReport()` produces 30-day trends with trajectory analysis
- Users can query `status` (full report) or `streak` (per-supplement breakdown) at any time

### Safety Layer

**Interaction checker** — runs on every `registerUser()` call. Checks the user's full stack (supplements + peptides) against 18 known interactions. Warnings are sent to the user before their first reminder.

Known interactions include: Magnesium ↔ Zinc absorption competition, Fish Oil + high-dose Vitamin E bleeding risk, Berberine + Metformin hypoglycemia potentiation, NAD+ precursors + SIRT1 activators (dose timing note), and 14 others.

**Side effect handler** — `classifySideEffect()` classifies severity via regex before any LLM call:
- `emergency` (chest pain, difficulty breathing, anaphylaxis) → immediate 911 message, no AI delay
- `severe` (severe allergic reaction, extreme dizziness) → urgent escalation
- `moderate` → AI analysis with dose correlation
- `mild` → standard AI analysis

### Multi-Step Onboarding

Unknown senders who message "start", "hi", "hello", or "join" are offered onboarding. The `OnboardingService` runs a 4-step conversation flow:

1. Name
2. Goals (longevity / sleep / skin / energy / cognition / body-composition)
3. Wake time
4. Sleep time

State is held in `ConversationFlowManager` (in-memory, 30-minute TTL). On completion, a personalized AI-generated welcome message is sent.

### SDK Plugins (3 built-in)

| Plugin | Role |
|--------|------|
| `complianceLoggerPlugin` | Logs every outbound send with masked phone number to stdout |
| `safetyGuardPlugin` | Blocks sends containing dangerous content patterns (blocks the send, logs the violation) |
| `aiGenerationPlugin` | Pass-through hook for future pre-send enrichment |

### Multi-User Platform

Every feature is scoped per user:
- Each user has their own row in the `users` table
- Reminders are scheduled per user with their timezone and wake/sleep times
- Protocol activation is per-user based on their goals
- Compliance logs are keyed by `userId`
- Conversation flow state is keyed by phone number

### Message Flow

```
iMessage inbox
      │
      ▼
  SDK watcher (WAL-based real-time detection)
      │
      ▼
  sdk.watch(callback)
      │
      ├─ Unknown sender? → check if onboarding trigger → OnboardingService
      │
      ├─ Active conversation flow? → OnboardingService.handle()
      │
      └─ Registered user → routeCommand(text)
              │
              ├─ compliance_taken → logCompliance(taken) + encouragement
              ├─ compliance_skip  → logCompliance(skip) + understanding reply
              ├─ status           → ReportingService → generateReport() → Claude analysis
              ├─ streak           → calculateStreak() → formatStreakSummary()
              ├─ reconstitute     → buildReconstitutionGuide() → Claude Q&A
              ├─ side_effect      → classifySideEffect() → escalate or Claude analysis
              ├─ skin_checkin     → buildSkinCheckInPrompt() → Claude coaching
              ├─ sleep_checkin    → buildSleepCheckInPrompt() → Claude coaching
              ├─ protocol_list    → buildProtocolListMessage()
              ├─ adjustment       → Claude + flag for manual review
              ├─ help             → buildHelpMessage()
              └─ general          → Claude (full health Q&A)
```

### SQLite Schema

```sql
users (
  id TEXT PRIMARY KEY,
  phone TEXT UNIQUE,
  name TEXT,
  profile_json TEXT,      -- full UserProfile serialized
  created_at INTEGER,
  onboarded INTEGER DEFAULT 0
)

compliance_log (
  id TEXT PRIMARY KEY,
  user_id TEXT REFERENCES users(id),
  supplement_name TEXT,
  taken INTEGER,          -- 1 = taken, 0 = skipped
  note TEXT,
  logged_at INTEGER
)

peptide_injections (
  id TEXT PRIMARY KEY,
  user_id TEXT REFERENCES users(id),
  peptide_name TEXT,
  dose_mcg REAL,
  site TEXT,
  logged_at INTEGER
)
```

## Setup

### Requirements

- **macOS** (iMessage SDK requirement — reads `~/Library/Messages/chat.db`)
- **Bun >= 1.0.0**
- **Full Disk Access** granted to Terminal (System Settings → Privacy & Security → Full Disk Access)
- An **Anthropic API key**

### Install

```bash
# From repo root
bun install

# Configure environment
cp app/.env.example app/.env
```

Edit `app/.env`:

```env
ANTHROPIC_API_KEY=sk-ant-...
DB_PATH=./data/healthspan.db
DEBUG=false
DEFAULT_TIMEZONE=America/Los_Angeles
ADMIN_PHONE=+1234567890
```

### Register Users

```typescript
import { HealthspanOS } from './src/application/healthspan-os.ts'

await app.registerUser({
    phone: '+1234567890',
    name: 'Alex',
    timezone: 'America/Los_Angeles',
    wakeTime: '07:00',
    sleepTime: '22:30',
    goals: ['longevity', 'sleep', 'body-composition'],
    preferences: {
        reminderFrequency: 'strict',
        aiTone: 'engaging',
        photoTracking: false,
        voiceJournal: false,
    },
    stack: [
        { name: 'Vitamin D3', dose: '5000 IU', timing: 'morning', frequency: 'daily', purpose: 'Immune function', interactions: [] },
        { name: 'Magnesium Glycinate', dose: '400 mg', timing: 'bedtime', frequency: 'daily', purpose: 'Sleep quality', interactions: [] },
        { name: 'NMN', dose: '500 mg', timing: 'morning', frequency: 'daily', purpose: 'NAD+ precursor', interactions: [] },
    ],
    peptides: [
        {
            name: 'Ipamorelin',
            type: 'growth-hormone',
            doseMcg: 200,
            frequencyPerWeek: 5,
            injectionTime: '21:30',
            cycleWeeks: 12,
            restWeeks: 4,

            reconstitution: { vialMg: 5, bacWaterMl: 2, doseUnits: 8 },
            rotationSites: ['Left abdomen', 'Right abdomen', 'Left outer thigh', 'Right outer thigh'],
        },
    ],
})

await app.start()
```

### Start

```bash
cd app
bun start
```

## User Commands

Send any of these via iMessage to the number running Healthspan OS:

| Message | Response |
|---------|----------|
| `done` / `taken` / `yes` | Logs current supplement as taken, returns streak |
| `skip [reason]` / `no` | Logs missed dose, returns compassionate reply |
| `status` | 30-day compliance report with Claude trend analysis |
| `streak` | Per-supplement streak breakdown |
| `reconstitute [peptide]` | BAC water mixing guide + open AI Q&A |
| `side effects [symptoms]` | Severity classification, dose correlation, escalation if needed |
| `skin [rating/notes]` | Skin protocol check-in with Claude coaching |
| `sleep [score]` | Sleep quality check-in with circadian coaching |
| `protocols` | List active supplements, peptides, and health protocols |
| `adjust [what]` | Request a protocol change (flagged for review) |
| `help` | Full command reference |
| Anything else | General health Q&A via Claude |

### Self-Onboarding

New users can message `start`, `join`, or `sign up` to begin the 4-step onboarding flow. Healthspan OS will collect their name, goals, and schedule — then send an AI-generated personalized welcome message.
